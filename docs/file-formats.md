# File Formats

This page explains the lyric formats used by `lrc-tools`.

## `.lrc` — standard LRC

Each line has a timestamp (MM:SS.xx) and a phrase of text. Used by `lrc-fetch`.

### Example

```
[00:12.00]Hello darkness, my old friend
[00:17.20]I've come to talk with you again
[00:22.10]Because a vision softly creeping
[00:27.40]Left its seeds while I was sleeping
```

### Metadata tags

Tags like `[ti:Title]`, `[ar:Artist]`, `[al:Album]` are parsed by `parse_metadata()`.

## `.wlrc` — word-level LRC

Each timestamp maps to a single word. Generated by `lrc-processor --wlrc`. Used by `lrc-vis --wlrc` for word-by-word playback.

### Example

```
[00:12.00]Hello
[00:12.45]darkness,
[00:13.10]my
[00:13.40]old
[00:13.80]friend
[00:17.20]I've
```

## `.srt` — SubRip subtitles

Exported from `.lrc` via `exporters.export_srt()`. Standard subtitle format playable in video players.

### Example

```
1
00:00:01,000 --> 00:00:03,500
Hello darkness, my old friend

2
00:00:03,500 --> 00:00:07,200
I've come to talk with you again
```

## `.json` — JSON export

Portable structured format exported from `.lrc` via `exporters.export_json()`. Includes metadata and ordered lyric lines.

### Example

```json
{
  "metadata": { "ti": "Sound of Silence", "ar": "Simon & Garfunkel" },
  "source": "/path/to/file.lrc",
  "lyrics": [
    { "timestamp": 12.0, "text": "Hello darkness, my old friend" }
  ]
}
```

## Format usage

| Operation | Input | Output |
|-----------|-------|--------|
| `lrc-fetch` | Audio files | `.lrc` |
| `lrc-processor` | `.lrc` | `.lrc` or `.wlrc` |
| `lrc-vis` | `.wlrc` or `.lrc` | Terminal display |
| `export_srt()` | `.lrc` | `.srt` |
| `export_json()` | `.lrc` | `.json` |
| `import_srt()` | `.srt` | dict list (for `write_lrc`) |
| `import_json()` | `.json` | dict list (for `write_lrc`) |

## Matching rules

The lyric filename should match the audio filename stem:

```
Artist - Song.mp3     ← audio
Artist - Song.lrc     ← lyrics
Artist - Song.wlrc    ← word-level lyrics
Artist - Song.srt     ← subtitles
```

Metadata tags inside the file are also used for matching (when mutagen is available).
